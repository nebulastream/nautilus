# Nautilus Runtime Options

Nautilus provides various runtime options to customize the compilation. Options come in two scopes:

- **Engine** – passed when initializing the Nautilus engine (`NautilusEngine(options)`). They configure the
  engine-owned, built-once compiler and therefore apply to every module compiled by that engine. Engine-only options
  cannot be overridden per module.
- **Module** – every option below can be set on the engine to provide an engine-wide default. The options marked
  *Module* in the **Scope** column can additionally be **overridden for an individual compiled module**, either by
  passing a `ModuleOptions` to `engine.createModule(overrides)` or by calling `module.setOption(name, value)` before
  `module.compile()`. A module inherits all engine-wide values and overrides only what it sets.

See [engine.md](engine.md) for how to create modules and override options per module.

| Option                                      | Default  | Scope   | Description                                                                                                                                           |
|---------------------------------------------|----------|---------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| engine.compilation=[true,false]             | true     | Engine  | Activates the compilation of nautilus functions. If the flag is false, all nautilus functions will be directly invoked, which is handy for debugging. |
| engine.backend=[mlir,cpp,bc,tbc,asmjit]     | -        | Engine  | Pins a single compilation backend: the engine compiles single-tier, synchronously, with exactly this backend (no tier-0 compile, no background promotion). When unset, the engine runs the default tiered compilation. |
| engine.tier0.backend / engine.tier1.backend | see desc. | Engine  | Configures the fast (tier 0) and optimized (tier 1) backends for tiered compilation. Both must be set to override the defaults; ignored when `engine.backend` is set. By default tier 1 is `mlir` and tier 0 is selected by availability in the order `asmjit`, `bc`, `interpreter`. Tier 0 also accepts the special value `interpreter`: the module runs uncompiled (direct invocation) until the background tier-1 compilation completes. |
| engine.tiered.backgroundPromotion=[true,false] | true  | Engine  | When true, the engine compiles with the tier-0 backend immediately and promotes to the tier-1 backend in the background. When false, it compiles single-tier with the tier-1 backend (synchronous, no promotion). |
| engine.normalizeFunctionNames=[true,false]  | false    | Module  | Generates normalized function names (runtimeFunc0, runtimeFunc1, etc.) during tracing instead of using actual function names. Useful for deterministic output and testing. |
| engine.demangleFunctionNames=[true,false]   | true     | Module  | Demangles C++ function names to human-readable format during tracing. When disabled, uses mangled names as-is. Ignored if normalizeFunctionNames is enabled. |
| engine.logStatistics=[true,false]           | false    | Module  | Logs compilation statistics (timings, entity counts) after compilation.                                                                              |
| dump.all=[true,false]                       | false    | Module  | Dumbs intermediate representations all compilation steps.                                                                                             |
| dump.after_tracing=[true,false]             | false    | Module  | Dumbs traces directly after trace generation.                                                                                                         |
| dump.after_ssa=[true,false]                 | false    | Module  | Dumbs traces after SSA generation.                                                                                                                    |
| dump.after_ir_creation=[true,false]         | false    | Module  | Dumbs the nautilus ir after generation.                                                                                                               |
| dump.after_mlir_generation=[true,false]     | false    | Module  | Dumbs the generated mlir if the MLIR backend is used.                                                                                                 |
| dump.after_llvm_generation=[true,false]     | false    | Module  | Dumbs the generated llvm if the MLIR backend is used.                                                                                                 |
| dump.after_cpp_generation=[true,false]      | false    | Module  | Dumbs the generated cpp code if the CPP backend is used.                                                                                              |
| dump.after_bc_generation=[true,false]       | false    | Module  | Dumbs the generated byte codes if the Bytecode Interpreter (BC) backend is used.                                                                      |
| dump.console=[true,false]                   | false    | Module  | Enables the dumping of intermediate compilation steps to the console.                                                                                 |
| dump.file=[true,false]                      | true     | Module  | Enables the dumping of intermediate compilation steps to a temp folder.                                                                               |
| dump.graph=[true,false]                     | false    | Module  | Dumps a graph representation in either (mermaid or graphvit) to a file.                                                                               |
| dump.graph.type=[mermaid,graphviz]          | graphviz | Module  | Sets the type of the graph.                                                                                                                           |
| dump.graph.full=[true,false]                | false    | Module  | Represents both data and control-flow (full=true) or only the control-flow (full=false).                                                              |
| ir.runPasses=[true,false]                   | true     | Module  | Runs the IR optimization pass pipeline.                                                                                                               |
| ir.disableDeadCodeElimination=[true,false]  | false    | Module  | Disables the pass that removes pure, unused operations (constants, casts, arithmetic) from the IR.                                                    |
| ir.disableAlgebraicSimplification=[true,false] | false | Module  | Disables local algebraic simplifications (identities, bitwise/boolean folds, cast collapsing, constant-to-right canonicalization) in the IR.          |
| ir.disableConstantBranchFolding=[true,false] | false   | Module  | Disables folding `if`s with a compile-time-constant or duplicate-arm condition into unconditional branches and sweeping the resulting unreachable blocks. |
| ir.disableBlockMerging=[true,false]         | false    | Module  | Disables merging blocks that are the sole target of an unconditional branch into their predecessor (collapsing single-predecessor chains).            |
| ir.disableBlockArgumentPruning=[true,false] | false    | Module  | Disables removing block arguments that are unused or receive the identical, dominating value from every predecessor (entry-block arguments are never touched). |
| ir.enableLocalCSE=[true,false]              | false    | Module  | Enables block-local common-subexpression elimination over computed pure operations (opt-in pending a benchmark sweep before it becomes default-on). Loads and constants are never CSE'd. |
| ir.enableLICM=[true,false]                  | false    | Module  | Enables hoisting loop-invariant pure operations into a natural loop's preheader (opt-in; loads and possibly-trapping divisions are never hoisted).    |
| mlir.optimizationLevel=[0,1,2,3]            | 3        | Module  | Sets the optimization level for the code-generation if the MLIR backend is used.                                                                      |
| mlir.enableMultithreading=[true,false]      | true     | Module  | Allows MLIR Backend to use multiple threads.                                                                                                          |
| mlir.enableIntrinsics=[true,false]          | true     | Module  | Registers the MLIR intrinsic plugins for this compilation.                                                                                            |
| mlir.inline_invoke_calls=[true,false]       | false    | Module  | Allow MLIR Backend to inline functions that were tagged with NAUTILUS_INLINE                                                                          |
| bc.registerAllocator=[true,false]           | true     | Module  | Enables the linear register allocator in the bytecode backend.                                                                                        |
| tbc.mode=[interp,jit,auto]                  | interp   | Module  | Execution mode of the tbc backend: `interp` runs the dispatch-loop interpreter; `jit` stitches copy-and-patch native code (throws when this build cannot execute it: needs a generated stencil table for the target, a Clang 19+ build, no ASan); `auto` uses the JIT when available and degrades to the interpreter. |
| tbc.dispatch=[auto,tailcall,goto,switch]    | auto     | Module  | Interpreter dispatch skin of the tbc backend; `auto` picks the strongest the build supports (tail-call threading on Clang, computed goto on GCC, switch elsewhere). Unavailable requests degrade silently.            |
| tbc.stackSizeKb                             | 1024     | Module  | Size of the tbc backend's per-thread VM stack in KiB.                                                                                                 |
| tbc.registerAllocator=[true,false]          | true     | Module  | Enables the linear register allocator in the tbc backend's lowering.                                                                                  |
| tbc.coalescing=[true,false]                 | true     | Module  | Emits minimal MOV sequences for block-edge parallel copies in the tbc backend.                                                                        |
| tbc.superinstructions=[true,false]          | true     | Module  | Fuses single-use comparisons into two-word compare-and-branch instructions in the tbc backend.                                                        |
| tbc.immediates=[true,false]                 | true     | Module  | Folds small constant operands into 16-bit immediate fields in the tbc backend.                                                                        |
| asmjit.enableIntrinsics=[true,false]        | true     | Module  | Registers the AsmJit intrinsic plugins for this compilation.                                                                                          |
| asmjit.enablePostRAPeephole=[true,false]    | true     | Module  | Enables post-register-allocation peephole optimizations in the AsmJit backend.                                                                        |
| asmjit.enableBranchFusion=[true,false]      | true     | Module  | Fuses a compare into the conditional branch that is its only consumer (cmp+jcc instead of cmp+setcc+movzx+test+jz) in the AsmJit x86-64 backend.      |
| asmjit.enableConstFolding=[true,false]      | true     | Module  | Defers integer/bool/ptr constants in the AsmJit x86-64 backend; consumers fold them as immediate operands (add/sub/cmp/shift/imul/and/or/xor) or rematerialise per use. |
| asmjit.enableSelectCmov=[true,false]        | true     | Module  | Lowers integer selects branch-free via cmov in the AsmJit x86-64 backend (mirrors the A64 backend's csel).                                            |

> **Note:** The JIT event listeners registered via `Options::addMLIRJitEventListener` are an engine-scoped resource
> (their lifetime is tied to the engine) and are inherited by every module.
