# Nautilus Runtime Options

Nautilus provides various runtime options to customize the compilation. Options come in two scopes:

- **Engine** – passed when initializing the Nautilus engine (`NautilusEngine(options)`). They configure the
  engine-owned, built-once compiler and therefore apply to every module compiled by that engine. Engine-only options
  cannot be overridden per module.
- **Module** – every option below can be set on the engine to provide an engine-wide default. The options marked
  *Module* in the **Scope** column can additionally be **overridden for an individual compiled module**, either by
  passing a `ModuleOptions` to `engine.createModule(overrides)` or by calling `module.setOption(name, value)` before
  `module.compile()`. A module inherits all engine-wide values and overrides only what it sets.

See [engine.md](engine.md) for how to create modules and override options per module.

| Option                                      | Default  | Scope   | Description                                                                                                                                           |
|---------------------------------------------|----------|---------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| engine.compilation=[true,false]             | true     | Engine  | Activates the compilation of nautilus functions. If the flag is false, all nautilus functions will be directly invoked, which is handy for debugging. |
| engine.backend=[mlir,bc,cpp]                | mlir     | Engine  | Sets the specific compilation backend for nautilus functions.                                                                                         |
| engine.compilationStrategy=[tiered,legacy]  | tiered   | Engine  | Selects the compilation strategy. Used only when `engine.backend` is not set.                                                                         |
| engine.tier0.backend / engine.tier1.backend | -        | Engine  | Configures the fast (tier 0) and optimized (tier 1) backends for tiered compilation.                                                                  |
| engine.normalizeFunctionNames=[true,false]  | false    | Module  | Generates normalized function names (runtimeFunc0, runtimeFunc1, etc.) during tracing instead of using actual function names. Useful for deterministic output and testing. |
| engine.demangleFunctionNames=[true,false]   | true     | Module  | Demangles C++ function names to human-readable format during tracing. When disabled, uses mangled names as-is. Ignored if normalizeFunctionNames is enabled. |
| engine.logStatistics=[true,false]           | false    | Module  | Logs compilation statistics (timings, entity counts) after compilation.                                                                              |
| dump.all=[true,false]                       | false    | Module  | Dumbs intermediate representations all compilation steps.                                                                                             |
| dump.after_tracing=[true,false]             | false    | Module  | Dumbs traces directly after trace generation.                                                                                                         |
| dump.after_ssa=[true,false]                 | false    | Module  | Dumbs traces after SSA generation.                                                                                                                    |
| dump.after_ir_creation=[true,false]         | false    | Module  | Dumbs the nautilus ir after generation.                                                                                                               |
| dump.after_mlir_generation=[true,false]     | false    | Module  | Dumbs the generated mlir if the MLIR backend is used.                                                                                                 |
| dump.after_llvm_generation=[true,false]     | false    | Module  | Dumbs the generated llvm if the MLIR backend is used.                                                                                                 |
| dump.after_cpp_generation=[true,false]      | false    | Module  | Dumbs the generated cpp code if the CPP backend is used.                                                                                              |
| dump.after_bc_generation=[true,false]       | false    | Module  | Dumbs the generated byte codes if the Bytecode Interpreter (BC) backend is used.                                                                      |
| dump.console=[true,false]                   | false    | Module  | Enables the dumping of intermediate compilation steps to the console.                                                                                 |
| dump.file=[true,false]                      | true     | Module  | Enables the dumping of intermediate compilation steps to a temp folder.                                                                               |
| dump.graph=[true,false]                     | false    | Module  | Dumps a graph representation in either (mermaid or graphvit) to a file.                                                                               |
| dump.graph.type=[mermaid,graphviz]          | graphviz | Module  | Sets the type of the graph.                                                                                                                           |
| dump.graph.full=[true,false]                | false    | Module  | Represents both data and control-flow (full=true) or only the control-flow (full=false).                                                              |
| ir.runPasses=[true,false]                   | true     | Module  | Runs the IR optimization pass pipeline.                                                                                                               |
| mlir.optimizationLevel=[0,1,2,3]            | 3        | Module  | Sets the optimization level for the code-generation if the MLIR backend is used.                                                                      |
| mlir.enableMultithreading=[true,false]      | true     | Module  | Allows MLIR Backend to use multiple threads.                                                                                                          |
| mlir.enableIntrinsics=[true,false]          | true     | Module  | Registers the MLIR intrinsic plugins for this compilation.                                                                                            |
| mlir.inline_invoke_calls=[true,false]       | false    | Module  | Allow MLIR Backend to inline functions that were tagged with NAUTILUS_INLINE                                                                          |
| bc.registerAllocator=[true,false]           | true     | Module  | Enables the linear register allocator in the bytecode backend.                                                                                        |
| asmjit.enableIntrinsics=[true,false]        | true     | Module  | Registers the AsmJit intrinsic plugins for this compilation.                                                                                          |
| asmjit.enablePostRAPeephole=[true,false]    | true     | Module  | Enables post-register-allocation peephole optimizations in the AsmJit backend.                                                                        |

> **Note:** The JIT event listeners registered via `Options::addMLIRJitEventListener` are an engine-scoped resource
> (their lifetime is tied to the engine) and are inherited by every module.
